## 共通点

- 1変数方程式 `f(x) = 0` をニュートン法で解く
- 多変数連立方程式 `F(x) = 0` をニュートン法で解く
- 収束判定には関数値および更新量の大きさを使う（閾値 `tol = 1e-10`）
- テスト用の非線形方程式を用意し，計算した解と真の解を比較する
- 浮動小数点演算は**倍精度**（64 bit, 機械イプシロン ≈ 2.22×10⁻¹⁶）

---

## ニュートン法のアルゴリズム

### 1 変数（スカラー方程式）

$$x_{k+1} = x_k - \frac{f(x_k)}{f'(x_k)}$$

**収束判定**：$\lvert f(x_k)\rvert < \varepsilon$ かつ $\lvert\Delta x_k\rvert < \varepsilon$

### N 変数（連立方程式）

各ニュートンステップで次の線形方程式を解く。

$$J(\mathbf{x}_k)\,\Delta\mathbf{x}_k = -F(\mathbf{x}_k), \qquad \mathbf{x}_{k+1} = \mathbf{x}_k + \Delta\mathbf{x}_k$$

$J(\mathbf{x})$ はヤコビアン行列：$J_{ij} = \partial F_i / \partial x_j$

**収束判定**：$\lVert F(\mathbf{x}_k)\rVert < \varepsilon$ かつ $\lVert\Delta\mathbf{x}_k\rVert < \varepsilon$

> ニュートン法は 2 次収束する。初期値が解の近傍にある場合，誤差 $e_k$ に対して
> $e_{k+1} \approx C\,e_k^2$ が成り立つ。

## ニュートン法の理論と数値計算への応用

### 1変数の理論：導出と収束性

#### テイラー展開による導出

$f: \mathbb{R} \to \mathbb{R}$ が $C^2$ 級（2階連続微分可能）であるとする。
現在の近似解 $x_k$ の近傍で $f$ を 1 次のテイラー展開すると，

$$f(x_k + \Delta x) \approx f(x_k) + f'(x_k)\,\Delta x$$

右辺をゼロとおいて $\Delta x$ を求めると，

$$\Delta x = -\frac{f(x_k)}{f'(x_k)}$$

したがって，次の近似解の更新式が得られる。

$$\boxed{x_{k+1} = x_k - \frac{f(x_k)}{f'(x_k)}}$$

これがニュートン法の基本公式である。

#### 幾何学的解釈

```
 f(x)
  │         ／
  │        ／  ← 接線 y = f(x_k) + f'(x_k)(x - x_k)
  │       ／
  │      ／
  │  ●──────────────────── x 軸との交点 = x_{k+1}
  │ f(x_k)
  │
──┼─────────────┬──────── x
                x_{k+1}  x_k
```

> 点 $(x_k, f(x_k))$ における接線と $x$ 軸の交点が次の近似解 $x_{k+1}$ となる。
> 関数が十分滑らかで初期値が解の近傍にあれば，反復ごとに誤差が急激に縮小する。

#### 局所 2 次収束定理

$x^{\ast}$を $f(x^{\ast}) = 0$ の単純根（$f'(x^{\ast}) \neq 0$）とする。
$f$ が $x^{\ast}$ の近傍で $C^2$ 級であれば，初期値 $x_0$ が $x^{\ast}$ に十分近い場合，
ニュートン法は **2 次収束** する：

$$\lvert x_{k+1} - x^{\ast}\rvert \leq C\,\lvert x_k - x^{\ast}\rvert^2, \qquad C = \frac{\lvert f''(x^{\ast})\rvert}{2\lvert f'(x^{\ast})\rvert}$$

ここで $e_k = \lvert x_k - x^{\ast}\rvert$ を誤差とすると，$e_{k+1} \approx C\,e_k^2$ が成り立つ。

| 反復回数 | 有効桁数の目安（$C \approx 1$ のとき） |
|:--------:|:-----------------------------------:|
| 0 | 1 桁 |
| 1 | 2 桁 |
| 2 | 4 桁 |
| 3 | 8 桁 |
| 4 | 16 桁（倍精度の限界） |

> 2 次収束のため，有効桁数が**反復ごとに約 2 倍**になる。これが「高速収束」の直感的な意味。

#### 収束しない場合

ニュートン法が失敗する主なケースは以下の 3 つ。

| 状況 | 原因 | 対策 |
|:-----|:-----|:-----|
| 導関数がゼロに近い | $f'(x_k) \approx 0$（停留点付近） | ダンピング，別の初期値 |
| 初期値が解から遠い | 収束半径の外側 | ダンピング付きニュートン法，区間二分法で初期値推定 |
| 振動・発散 | 非凸関数でのオーバーシュート | ダンピング（ステップ幅制御） |

### 多変数への拡張

#### テイラー展開による導出（N 変数）

$F: \mathbb{R}^n \to \mathbb{R}^n$ が $C^2$ 級であるとする。
$\mathbf{x}_k$ 周りの 1 次テイラー展開は，

$$F(\mathbf{x}_k + \Delta\mathbf{x}) \approx F(\mathbf{x}_k) + J(\mathbf{x}_k)\,\Delta\mathbf{x}$$

ここで $J(\mathbf{x}) \in \mathbb{R}^{n \times n}$ は**ヤコビアン行列**：

$$J_{ij}(\mathbf{x}) = \frac{\partial F_i}{\partial x_j}(\mathbf{x})$$

右辺をゼロとおくと線形方程式が得られる：

$$J(\mathbf{x}_k)\,\Delta\mathbf{x}_k = -F(\mathbf{x}_k)$$

この線形方程式を $\Delta\mathbf{x}_k$ について解いて更新する：

$$\boxed{\mathbf{x}_{k+1} = \mathbf{x}_k + \Delta\mathbf{x}_k}$$

> 実装上は $J^{-1}$ を直接計算せず，線形方程式として解くことで効率化・安定化を図る。

#### ヤコビアン行列の具体例

2 変数の場合 $F = (F_1, F_2)^T$，$\mathbf{x} = (x_1, x_2)^T$：

$$J(\mathbf{x}) = \begin{pmatrix} \dfrac{\partial F_1}{\partial x_1} & \dfrac{\partial F_1}{\partial x_2} \\ \dfrac{\partial F_2}{\partial x_1} & \dfrac{\partial F_2}{\partial x_2} \end{pmatrix}$$

各ニュートンステップで解く線形方程式：

$$\begin{pmatrix} \partial_{x_1}F_1 & \partial_{x_2}F_1 \\ \partial_{x_1}F_2 & \partial_{x_2}F_2 \end{pmatrix} \begin{pmatrix} \Delta x_1 \\ \Delta x_2 \end{pmatrix} = -\begin{pmatrix} F_1(\mathbf{x}_k) \\ F_2(\mathbf{x}_k) \end{pmatrix}$$

#### 多変数の収束性

1 変数と同様に，$\mathbf{x}^{\ast}$ の近傍で 2 次収束が成立する：

$$\lVert\mathbf{x}_{k+1} - \mathbf{x}^{\ast}\rVert \leq C\,\lVert\mathbf{x}_k - \mathbf{x}^{\ast}\rVert^2$$

条件：$F(\mathbf{x}^{\ast}) = \mathbf{0}$，$J(\mathbf{x}^{\ast})$ が正則（$\det J \neq 0$），$F$ が $C^2$ 級。

### 数値計算への応用

#### 数値微分によるヤコビアン近似

解析的なヤコビアンが手に入らない場合，**中心差分**でヤコビアンの各列を近似する。

$$\frac{\partial F_i}{\partial x_j}(\mathbf{x}) \approx \frac{F_i(\mathbf{x} + h\,\mathbf{e}_j) - F_i(\mathbf{x} - h\,\mathbf{e}_j)}{2h}$$

ここで $\mathbf{e}_j$ は $j$ 番目の単位ベクトル。

**刻み幅 $h$ の選択**：

$$h = \sqrt{\varepsilon_{\text{mach}}} \cdot \max(1,\, \lvert x_j\rvert), \qquad \varepsilon_{\text{mach}} \approx 2.22 \times 10^{-16}$$

| 刻み幅の効果 | 小さすぎる $h$ | 大きすぎる $h$ | 最適 $h \sim \varepsilon_{\text{mach}}^{1/2}$ |
|:------------|:------------:|:-------------:|:-------------------------------------------:|
| 打ち切り誤差 $O(h^2)$ | ← 小 | → 大 | バランス点 |
| 丸め誤差 $O(\varepsilon_{\text{mach}}/h)$ | → 大 | ← 小 | バランス点 |
| **合計誤差** | 大 | 大 | **最小 $\sim \varepsilon_{\text{mach}}^{1/2}$** |

> 前進差分 $[f(x+h)-f(x)]/h$ は $O(h)$ の精度しかないが，
> 中心差分は $O(h^2)$ の精度を持ち，最適 $h$ では数値微分誤差が $\sim 10^{-8}$ 程度になる。

**数値微分のコスト**：n 変数のヤコビアンを中心差分で構築するには $\mathbf{2n}$ 回の $F$ 評価が必要。
これが解析的ヤコビアンと比べて計算時間が大きくなる主因（実測で 7〜8 倍差）。

```
ヤコビアン計算コスト（1 ニュートンステップあたり）

解析的         │ F 評価  1 回 + J 計算  1 回
数値微分(中心) │ F 評価 2n 回（n 変数）
自動微分       │ F 評価  1 回分相当（計算グラフの自動構築）
```

#### 自動微分（ForwardDiff）

**ForwardDiff.jl** は数値微分ではなく，**双対数（dual number）** を用いた機械精度の微分を実現する。

通常の実数 $x$ の代わりに $x + \dot{x}\,\epsilon$（$\epsilon^2 = 0$）を伝播させることで，
関数評価と同時に正確な導関数値を得る。

$$f(x + \epsilon) = f(x) + f'(x)\,\epsilon$$

- **誤差のオーダー**：0（丸め誤差のみ，打ち切り誤差なし）
- **コスト**：ヤコビアンの列数だけ前向きパスを実行（$O(n)$ 回の $F$ 評価相当）
- **精度**：解析的ヤコビアンと等価（機械イプシロン級）

| 手法 | 精度 | コスト | 実装の手間 |
|:-----|:----:|:------:|:----------:|
| 解析的 | 機械精度 | $O(1)$ × $F$評価 | 手動で導出が必要 |
| 数値微分（中心差分） | $O(h^2) \sim 10^{-8}$ | $2n$ × $F$評価 | 不要（自動） |
| 自動微分（ForwardDiff） | 機械精度 | $\sim n$ × $F$評価 | 不要（自動） |

#### 線形方程式ソルバー（LU 分解）

各ニュートンステップで $J \Delta\mathbf{x} = -F(\mathbf{x})$ を解く必要がある。
これを**LU 分解**で解く。

**手順**（部分ピボット選択付き）：

1. **LU 分解**：$PA = LU$（$P$ は行置換行列，$L$ は下三角，$U$ は上三角）
2. **前進代入**：$L\mathbf{y} = P(-F)$ を解く
3. **後退代入**：$U\Delta\mathbf{x} = \mathbf{y}$ を解く

**計算量**：

$$\text{LU 分解: } O(n^3), \quad \text{前進・後退代入: } O(n^2)$$

```
LU 分解の計算量スケーリング

n=2:   ■
n=4:   ████████
n=8:   ████████████████████████████████████████████████████████████████
n=10:  ■■■■■■■■■■■■■■■■■■■■■■■■■■ (2³=8, 4³=64, 8³=512, 10³=1000)
```

> **ピボット選択の重要性**：係数行列 $J$ の特定行が数値的に小さい場合，
> ピボット選択なしのガウス消去では桁落ちが生じる。
> 部分ピボット選択（行の絶対値最大要素でピボットを選ぶ）により数値安定性を保証する。

**Fortran 版の実装**：自前 `lu_solve` サブルーチンで部分ピボット選択付き LU 分解を実装。
**Julia 版の実装**：`\` 演算子が LAPACK の `dgesv_` を呼び出し，キャッシュ最適化された LU 分解を実行。

#### ダンピング付きニュートン法（直線探索）

初期値が解から離れている場合，通常のニュートンステップ（$\alpha = 1$）で更新量が大きくなりすぎ，
発散することがある。**ダンピング**（減衰付き）ニュートン法では更新量を制御する。

$$\mathbf{x}_{k+1} = \mathbf{x}_k + \alpha_k\,\Delta\mathbf{x}_k, \qquad \alpha_k \in (0, 1]$$

$\alpha_k$ の選択（バックトラック法）：

```
alpha = 1.0
while ||F(x + alpha * dx)|| >= ||F(x)||:
    alpha = alpha / 2      ← ステップ幅を半分に
    if alpha < alpha_min:  ← 最小値以下なら打ち切り
        break
```

これは **Armijo 条件**（単純な減少条件）の一種であり，
$\alpha = 1$ から始めてステップ幅を半分にしながら $\lVert F\rVert$ が減少するまで探索する。

```
通常のニュートン法         ダンピング付きニュートン法
 f(x)                        f(x)
  │                             │
  │   ●  ← 発散                │
  │  ╱                          │       ● ← 安全な更新
  │ ╱                           │      ╱
  │╱                            │─────●
──┼────────── x              ──┼────────── x
  x_k   x_{k+1}               x_k x_{k+1}
 （オーバーシュート）          （alpha<1 で制御）
```

#### 収束判定の設計

数値計算における収束判定は，問題の特性に応じて適切に設計する必要がある。

**絶対残差基準**（本実装で採用）：

$$\lVert F(\mathbf{x}_k)\rVert < \varepsilon \quad \text{かつ} \quad \lVert\Delta\mathbf{x}_k\rVert < \varepsilon$$

- $\lVert F(\mathbf{x}_k)\rVert < \varepsilon$：方程式の残差が小さい（解の近くにいる）
- $\lVert\Delta\mathbf{x}_k\rVert < \varepsilon$：更新量が小さい（これ以上動かない）
- 両方を満たすことで**偽収束**（残差は小さいが解ではない）を防ぐ

| 判定基準 | 長所 | 短所 |
|:--------|:-----|:-----|
| 残差のみ $\lVert F\rVert < \varepsilon$ | シンプル | 更新量が大きくても収束と判定しうる |
| 更新量のみ $\lVert\Delta\mathbf{x}\rVert < \varepsilon$ | 解の変化を直接見る | ヤコビアンが特異に近い場合に誤検出 |
| **両方の AND**（本実装）| 誤検出を減らせる | やや保守的 |

**倍精度での実用的な閾値**：

$$\varepsilon = 10^{-10} \quad \text{（本実装のデフォルト）}$$

機械イプシロン $\approx 2.22 \times 10^{-16}$ に対して十分大きく，かつ実用上十分小さい値。

#### 数値安定性と条件数

ヤコビアン行列 $J$ が悪条件（condition number が大きい）の場合，
線形方程式 $J\Delta\mathbf{x} = -F$ の解が不正確になる。

$$\kappa(J) = \lVert J\rVert \cdot \lVert J^{-1}\rVert$$

```
条件数 κ(J) の影響

κ ≈ 1       → 完全良条件。解の誤差 ≈ 機械精度 × ||b|| / ||A||
κ ≈ 10⁸     → 中程度。倍精度で 8 桁の精度が失われる
κ ≈ 10¹⁶    → 実質特異。解の信頼性がない
```

> 本実装の Fortran 版では，ピボット要素が $10^{-15}$ 未満のとき行列を「特異に近い」と判断して停止する。
> Julia 版の LAPACK 実装も同様の内部チェックを行う。

```
入力: f（または F），x0
         │
         ▼
   導関数/ヤコビアンの準備
  ┌──────────────────────┐
  │ 解析的（ユーザー指定）│
  │ 数値微分（中心差分）  │  ← 自動選択
  │ 自動微分（ForwardDiff）│
  └──────────────────────┘
         │
         ▼
  ニュートンステップ計算
  dx = J \ (-F(x))  ← Julia の \ (LAPACK LU)
         │
         ▼
  ダンピング（オプション）
  バックトラックで alpha を決定
  x ← x + alpha * dx
         │
         ▼
  収束判定 → 収束なら解を返す
```

#### Fortran 版

```
入力: f, df（必須），x0
         │
         ▼
  導関数/ヤコビアンを
  利用者が必ず提供
         │
         ▼
  ニュートンステップ計算
  lu_solve(J, -F(x)) → dx  ← 自前 LU 分解
         │
         ▼
  x ← x + dx  （ダンピングなし）
         │
         ▼
  収束判定 → 収束なら解を返す
```

### 精度の比較

#### 理論上の精度

| 手法 | ヤコビアン誤差のオーダー | 達成可能精度 |
|:-----|:----------------------|:------------|
| Fortran（解析的） | 0（厳密） | 機械イプシロン級（~10⁻¹⁵） |
| Julia 解析的 | 0（厳密） | 機械イプシロン級（~10⁻¹⁵） |
| Julia 自動微分（ForwardDiff） | 0（厳密） | 機械イプシロン級（~10⁻¹⁵） |
| Julia 数値微分（中心差分） | $O(h^2)$，$h=\sqrt{\varepsilon_{\text{mach}}}$ | ~10⁻¹⁵（条件数良好時） |

> 中心差分の刻み幅 $h = \sqrt{\varepsilon_{\text{mach}}} \cdot \max(1, \lvert x\rvert)$ を採用しているため，
> 丸め誤差と打ち切り誤差のバランスが最適化され，解析的手法と同等の精度が得られる。

#### 手法別 計算時間の概算コスト比較

各ニュートンステップのコスト（次元 n での漸近的オーダー）：

| 処理 | analytic | finite | forwarddiff | Fortran |
|:-----|:--------:|:------:|:-----------:|:-------:|
| 関数評価 | O(n) | O(n²) ×2n | O(n) | O(n) |
| ヤコビアン計算 | O(n²) | O(n²) ×2n | O(n²) | O(n²) |
| 線形方程式ソルバー | O(n³) [LAPACK] | O(n³) [LAPACK] | O(n³) [LAPACK] | O(n³) [自前 LU] |

> **数値微分のコスト**：n 変数の数値ヤコビアンには中心差分で 2n 回の関数評価が必要。
> これが `finite` と `forwarddiff`・`analytic` の速度差の主因。

---

#### Fortran 版の速度特性

Fortran 版は直接コンパイル実行のため，Julia の JIT コンパイルと異なる特性を持つ。

| 比較観点 | Julia 版（analytic） | Fortran 版 |
|:--------|:------------------:|:----------:|
| **実行形態** | JIT コンパイル（初回は遅い） | AOT コンパイル（実行時はすぐに最速） |
| **ヤコビアン** | ユーザー提供（analytic と同等） | ユーザー提供（必須） |
| **線形ソルバー** | LAPACK（高度最適化済み） | 自前 LU 分解（教育的実装） |
| **ループ最適化** | Julia コンパイラによる自動最適化 | gfortran による最適化 |
| **メモリ配置** | 列優先（Julia は列優先） | 列優先（Fortran は列優先） |

> **Fortran の LU 分解 vs LAPACK**：
> Fortran 版の `lu_solve` は部分ピボット選択付きガウス消去法の教育的実装。
> Julia の `\` が呼び出す LAPACK の `dgesv_` は同じアルゴリズムを SIMD・キャッシュ最適化した実装。
> 小規模（n≤10）では差は小さいが，大規模問題では LAPACK が顕著に速い。

## 使い分けの指針

### Julia 版が適している場合

- 導関数・ヤコビアンを自分で書きたくない（数値微分で自動補完）
- `ForwardDiff.jl` による機械精度の自動微分を使いたい
- ダンピングや収束履歴など高度な機能が必要
- ベンチマーク・性能評価・グラフ化まで一括して行いたい
- 素早くプロトタイピングしたい

### Fortran 版が適している場合

- ニュートン法のアルゴリズム（LU 分解・ヤコビアン更新）を一行一行確認したい
- 解析的ヤコビアンが既に手元にある
- 外部ライブラリなし・コンパイルするだけで動く環境が必要
- 数値計算コードを直接 Fortran で書いている既存プロジェクトに統合したい
- アルゴリズムの教育的な理解・検証が目的

### 総評

```
                    精度  │  速度  │  使いやすさ  │  拡張性
────────────────────────────────────────────────────────────
Julia（analytic）:  ◎    │  ◎    │  ○          │  ◎
Julia（finite）:    ◎    │  △    │  ◎          │  ◎
Julia（forwarddiff）: ◎  │  △    │  ◎          │  ◎
Fortran:            ◎    │  ◎    │  △          │  ○
────────────────────────────────────────────────────────────
精度：全手法で機械イプシロン級（~10⁻¹⁵）のため実質差なし
速度：analytic/Fortran は最速，数値微分系は n 増加で遅くなる
使いやすさ：Julia は自動微分があり，Fortran は導関数が必須
```
